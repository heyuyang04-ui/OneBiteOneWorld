# 📌 创意背景
> 说明这个创意从哪里来，想解决什么真实问题，为什么值得做。
**创意来源，为什么有这个创意，解决什么问题：**

去年底，团队成员翻看自己过去一年的外卖订单，发现了从未意识到的真相：他点了47次火锅、23次日料，但8-10月辣食消费几乎为零——那正是他压力最大、频繁胃痛的时期；晚餐时间从18:30悄悄滑到21:15；1月和6月出现两个"甜食峰值"（分别是失恋和连续加班后）。这个发现让我们意识到：每个人吃的东西，其实是一部自己都没读过的自传。而手机里散落着这部自传的全部素材——外卖订单、食物照片、支付记录——却从没有人把它们拼起来、读出来。

顺着这个观察深挖，我们发现三个环环相扣的系统性痛点：

1. **看不见自己**：饮食数据散落在外卖App、相册、支付记录中，从未被整合成对自己的认知。用户不知道自己吃了什么、反映了怎样的生活状态、口味在如何变化。从"数据"到"认知"之间缺一个翻译器。

2. **找不到同类**：小红书知道你喜欢看美食内容，大众点评知道你去过哪些餐厅，抖音知道你看过哪些吃播。但没有产品能告诉你：世界上有一个人，对香菜的恨和你一样强烈，对某道菜和你有完全相同的心动。传统社交停留在"内容兴趣"，从未触及"真实味觉行为"。

3. **读不懂城市**：一座城在吃什么、口味如何流动、饮食文化如何变迁——这些信息精确地隐藏在千万人的餐桌上。但城市管理者看抽样问卷，餐饮老板凭经验猜趋势，普通人只能看"必吃榜"。没有产品把"千万人的餐桌"变成可读的城市智能。

我们认为这不是一个记录工具能解决的问题，必须是一个 Agent 驱动的感知系统。原因有三：饮食数据体量巨大、维度复杂；需要持续理解而非一次性计算；价值必须在个人-社交-城市三层之间流动。而2026年，多模态大模型让菜品识别可用、LLM 让数据变叙事、向量数据库让味觉匹配成为工程现实——技术窗口和需求窗口同时打开，Agent 重构产品范式的时机到了。

**目标用户 / 使用对象：**

18-35岁，愿意用食物理解自己、渴望在味觉层面找到同类、对城市饮食文化有好奇心的年轻人。三个典型画像：（1）外卖"打工人"阿杰（26岁，程序员），靠外卖续命三年，想知道自己到底吃了什么、健不健康；（2）城市"流浪者"小雅（28岁，设计师），刚搬到新城市，不想看大众榜单，想知道"这座城市和我的口味最匹配的是什么"；（3）美食"探险家"大鹏（22岁，大学生），把吃当作探索世界的入口，想知道世界上有没有人和他拥有一样的味觉。

**使用场景：**

*针对【关注健康的年轻人】，在【每周健康反思】下，为了解决【不清楚自己饮食趋势和健康关联】的问题，提供【Agent 生成味觉周报，洞察口味变化与健康趋势】的能力。*

*针对【出差/旅行者】，在【异地餐饮选择】下，为了解决【不想看大众榜单，想要个性化推荐】的问题，提供【Agent 基于个人味觉档案精准推荐最匹配餐厅】的能力。*

*针对【渴望社交连接的年轻人】，在【美食社交】下，为了解决【传统社交太泛，找不到真正味觉同频的人】的问题，提供【Agent 基于脱敏味觉向量匹配味觉灵魂伴侣】的能力。*

# 🎯 解决方案
> 说明作品是什么，怎么解决问题，核心功能是什么。
**解决方案简介：**

**一食万象**（One Bite, One World）是一个 Agent 驱动的城市味觉感知系统。它不是又一个美食记录工具，而是帮你从每一顿饭中看见更大世界的智能伙伴——一箪食中照见个人生活轨迹，一味之间连接味觉同类，万千餐桌汇聚成城市级饮食智能。

产品核心是一个三层 Agent 价值网络：L1 个人味觉档案（Agent 持续学习构建动态味觉数字孪生，每周生成洞察周报，预测口味趋势）；L2 味觉社交网络（Agent 基于脱敏的味觉向量匹配灵魂伴侣，两个 Agent 互相对话推荐餐厅）；L3 城市味觉地图（聚合数据生成城市级口味热力图、趋势检测、公共健康预警）。

核心范式转变：传统产品是"人找内容"的被动工具，一食万象是"Agent 替人感知"的主动伙伴——从记录到理解，从工具到关系，从单一用户价值到三层数据飞轮。

**核心功能/价值：**

**L1 个人味觉档案**：拍照即记录，Agent 自动识别菜品与营养；持续学习用户口味偏好，构建动态味觉模型；每周生成味觉周报，分析口味演变、饮食健康度、情绪饮食关联；基于历史数据预测用户下一步可能喜欢的口味和餐厅。

**L2 味觉社交网络**：Agent 计算用户间味觉相似度（128维向量加权余弦相似度），推荐"味觉灵魂伴侣"；展示共同菜系、口味交集、差异趣味；两个用户的 Agent 互相对话，联合生成推荐；所有匹配基于差分隐私处理后的脱敏向量，不暴露任何具体饮食记录。

**L3 城市味觉地图**：聚合千万用户脱敏数据，生成城市级口味热力图——辣味东移？甜食年轻化？打工人午餐时间缩短？Agent 分析饮食文化变迁，为餐饮行业提供选址参考、菜单优化、精准营销，为公共健康部门提供预警。

**用户使用流程：**

1. *用户吃饭前拍照上传食物照片。*
2. *Agent 自动识别菜品、估算营养成分，更新用户的味觉档案。*
3. *Agent 在后台持续学习——分析口味趋势、发现模式变化、建模情绪-饮食关联。*
4. *Agent 每周主动推送味觉周报，不定期推送惊喜发现（匹配到同类、口味正在转变、推荐新餐厅）。*
5. *用户切换城市视角，查看城市味觉热力图；到陌生城市时获得基于自身味觉档案的精准餐厅推荐。*
6. *用户对洞察进行反馈（对/不对），Agent 根据反馈持续优化模型，越用越准确。*

**作品亮点：**

1. **范式颠覆——从"人找内容"到"Agent 替人感知"**：传统产品是被动工具，用户必须主动搜索、记录、浏览。一食万象的 Agent 主动学习、主动洞察、主动推送、主动连接，产品从 App 变为一个持续进化的智能伙伴。用户不需要意志力维持使用习惯——Agent 在后台安静工作，在合适的时候告诉你合适的事。

2. **从一到万——三层价值飞轮，产品价值指数级扩展**：一箪食照见个人品位档案，一味之间连接万千灵魂，万千餐桌汇聚成城市饮食智能。用户的每一次用餐数据同时服务自己、服务社交发现、服务城市感知，形成"数据→洞察→更多数据→更深洞察"的正向飞轮，三层价值互相增强。

3. **不可替代的 Agent 能力**：千万人味觉数据的实时分析、128维口味向量的时序建模、跨维度的模式发现与关联解释、个性化的洞察生成——这些能力是传统互联网产品（人工编辑、规则引擎、协同过滤推荐算法）完全无法实现的，是 Agent 架构的天然优势。

# 🤖 Agent / AI 能力说明
> 本季 Hackathon 重点关注 Agent 和 AI 对工作流、产品形态和个人能力的重塑。请重点说明作品中 AI / Agent 扮演了什么角色。
## 1. 使用了哪些 AI / Agent 能力？

* *大模型理解 / 生成：LLM 将量化饮食数据转化为用户友好的叙事（周报、洞察文案），两个用户的 Agent 通过 LLM 互相对话生成联合推荐。*
* *Agent 任务规划：Orchestrator 根据事件类型（餐食上传、定位变化、定时触发）自动判断需要调用哪些 Agent 并分发任务。*
* *工具调用：Agent 调用图像识别模型、向量数据库、时序分析引擎、LLM 等多个工具和系统。*
* *多 Agent 协作：三个独立 Agent（个人Agent/社交Agent/城市Agent）各有感知-决策-行动闭环，通过统一消息协议协作。*
* *自动化执行：Cron 定时任务自动触发周报生成、匹配池更新、城市味觉快照预聚合。*
* *数据分析：时序编码器构建动态味觉向量，PELT 算法检测城市味觉趋势的结构性拐点。*
* *内容生成：LLM Chain（数据总结→模式发现→文案生成）自动生成周报、匹配解释、城市洞察报告。*
* *人机协同：用户拍照→Agent 识别→用户反馈→Agent 更新模型，形成持续学习闭环。*

## 2. Agent 在作品中承担什么角色？

|*Agent 角色*|*是否使用*|*说明*|
|-|-|-|
|*助手*|*是*|*帮助用户完成菜品识别、营养估算、信息整理等基础任务*|
|*分析师*|*是*|*分析用户饮食数据提炼个人洞察；分析城市聚合数据生成趋势报告；分析味觉向量进行社交匹配*|
|*执行者*|*是*|*自动触发周报生成、推送通知、匹配更新、城市快照预聚合等*|
|*协作者*|*是*|*与用户共同完成味觉建模——用户提供数据和反馈，Agent 持续学习和优化*|
|*流程编排者*|*是*|*Orchestrator 拆解事件、路由任务、协调三个 Agent 之间的协作*|

## 3. 和普通 AI 问答相比，新在哪里？

* *是否能完成连续任务？是。不是单次问答，而是"拍照→识别→建模→洞察→匹配→城市聚合"的完整连续链路，Agent 持续运行、持续学习。*
* *是否能主动拆解任务？是。Orchestrator 根据事件类型自动判断并路由，无需用户指令。例如上传火锅照片→同时触发个人Agent更新模型+城市Agent更新城市数据。*
* *是否能调用工具或系统？是。Agent 调用图像识别模型、向量数据库、时序分析引擎、LLM 等多个系统，形成完整技术栈整合。*
* *是否能连接业务流程？是。三层价值网络连接了个人健康管理、社交匹配、餐饮行业决策、公共健康预警等多个业务场景。*
* *是否能从输入到输出形成闭环？是。用户反馈→模型更新→更精准洞察→更好体验→更多数据，形成正向数据飞轮。*
* *是否有人机协同机制？是。用户是最终决策者，Agent 是持续学习和提案者。洞察可以反馈"对/不对"，匹配需双向确认，推荐可选采纳或忽略。*

# 📍 预期价值
> 说明该创意如果落地，预计能给业务带来什么价值。可以是效率、成本、体验、收入、质量、规模化推广等。
**对个人的价值**：让每一顿饭成为认识自己的入口。用户从"无意识地吃"变为"有意识地选择"，无需手动记录即可获得每周个性化洞察，提升饮食健康管理和自我认知。

**对社交的价值**：创造以味觉为纽带的深度连接方式。不同于兴趣标签的泛匹配，"味觉灵魂伴侣"基于真实的长期饮食行为数据，连接深度和信任感远超传统社交产品。

**对城市与商业的价值**：为餐饮行业提供数据智能基础设施——新店选址参考、菜单优化建议、精准营销投放。为公共健康部门提供实时饮食趋势预警。城市级数据报告可商业化，打开 2B/2G 市场。

**规模化潜力**：数据飞轮驱动增长——用户越多，城市洞察越强；城市洞察越强，吸引更多用户，形成不可替代的竞争壁垒。三层价值网络天然适合全球多城市部署。

**长期愿景**：当饮食数据跨越数年、覆盖千万人时，一食万象将成为理解社会变迁的文化基础设施——用食物记录一个时代的集体情绪与文化变迁。
