## 一、系统总览

### 1.1 核心设计理念

一食万象采用了 **Multi-Agent 协作架构**，将系统拆分为三个独立但协作的 Agent——个人Agent、社交Agent、城市Agent。每个 Agent 拥有独立的感知-决策-行动闭环，通过统一的编排层进行任务分发与结果聚合。

### 1.2 技术选型

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 端侧 | React Native / Flutter | 跨平台移动端，支持拍照、地理定位 |
| 网关 | GraphQL + WebSocket | 实时推送与查询解耦 |
| Agent 编排 | LangGraph / 自研 FSM 引擎 | 多 Agent 状态管理与任务路由 |
| 图像识别 | CLIP + 自研菜品分类模型 | 零样本菜品识别 + 营养估算 |
| 向量存储 | Milvus / pgvector | 味觉向量存储与相似度检索 |
| LLM | GPT-4o / 开源模型 | Agent 推理、周报生成、对话 |
| 时序分析 | Apache Druid / TimescaleDB | 城市级聚合查询 |
| 消息队列 | Kafka / Pulsar | 异步数据管线 |
| 数据湖 | Iceberg + MinIO | 脱敏数据存储与分析 |

## 三、Agent 详细设计

### 3.1 Agent 通用框架

每个 Agent 实现了统一的 `BaseAgent` 接口：

```python
class BaseAgent(ABC):
    """Agent 基类，定义统一的感知-决策-行动循环"""

    # ---------- 生命周期 ----------
    async def perceive(self, event: AgentEvent) -> Perception:
        """从事件中提取感知信息"""
        ...

    async def reason(self, perception: Perception, memory: AgentMemory) -> Decision:
        """基于感知+记忆进行推理决策"""
        ...

    async def act(self, decision: Decision) -> ActionResult:
        """执行决策并返回结果"""
        ...

    async def reflect(self, result: ActionResult, memory: AgentMemory) -> None:
        """基于结果更新记忆与模型"""
        ...

    # ---------- 记忆管理 ----------
    class AgentMemory:
        short_term: deque[MemoryItem]   # 短期记忆（最近 N 次交互）
        long_term: VectorStore          # 长期记忆（向量化存储）
        episodic: list[EpisodeSummary]  # 情节记忆（关键事件摘要）
        preference: TasteProfile        # 偏好模型（不断更新的用户味觉画像）
```

### 3.2 个人Agent（TasteAgent）

**职责**：构建用户味觉数字孪生，生成个性化洞察。

```
感知 ──→ 推理 ──→ 行动 ──→ 反思
  │        │        │        │
  │        │        │        └─ 更新味觉模型
  │        │        └─ 生成周报/推送洞察
  │        └─ LLM推理 + 规则引擎
  └─ 照片/行为/反馈
```

**核心能力模块**：

| 模块 | 输入 | 处理方法 | 输出 |
|------|------|---------|------|
| 菜品识别 | 食物照片 | CLIP Embedding → 分类器 → LLM验证 | 菜名、菜系、食材列表 |
| 营养估算 | 菜品识别结果 | 食谱数据库匹配 + 视觉体积估计 | 热量、蛋白质、碳水、脂肪 |
| 口味建模 | 菜品序列 × 反馈 | 协同过滤 + 时序编码器 + Online Learning | 128维味觉向量（动态更新） |
| 洞察生成 | 味觉向量 + 时序数据 + 上下文 | LLM Chain: 数据总结→模式发现→文案生成 | 周报、洞察卡片、餐食建议 |
| 趋势预测 | 历史味觉向量序列 | 时序Transformer + 生存分析 | 下一周/月的口味预测 |

**味觉向量设计**：

```
TasteVector (128-d):
├── 味型维度[0:31]      — 酸/甜/苦/辣/咸/鲜 × 强度 × 偏好
├── 菜系维度[32:63]     — 川/粤/湘/鲁/日/韩/东南亚/地中海 × 2
├── 食材维度[64:95]     — 常用食材的偏好嵌入
├── 场景维度[96:111]    — 工作餐/聚会/独食/约会/深夜 偏好
└── 时序维度[112:127]   — 周期性模式（周末 vs 工作日等）
```

**洞察生成 Prompt 链**：

```
Chain 1 — 数据总结：
"分析以下用户一周饮食数据，提取关键统计指标：
餐次 {meal_count}，高频菜系 {top_cuisines}，口味趋势 {taste_trend_7d}..."

Chain 2 — 模式发现：
"基于以上饮食数据及用户历史基线，发现显著变化：
- 相对于过去30天均值的偏离
- 是否存在周期性异常
- 情绪-饮食关联模式..."

Chain 3 — 文案生成：
"将以上分析转化为用户友好的周报。要求：温暖有洞察力，2-3个要点，
避免数据堆砌，给出一句总结性洞察。语气：朋友而非医生。"
```

### 3.3 社交Agent（SocialAgent）

**职责**：基于味觉向量进行匿名匹配与社交发现。

**味觉匹配算法**：

```python
class TasteMatcher:
    def compute_similarity(
        self,
        vector_a: TasteVector,   # 用户A的味觉向量
        vector_b: TasteVector,   # 用户B的味觉向量
        weights: MatchWeights    # 可调的匹配权重
    ) -> MatchResult:
        """
        多维度加权余弦相似度
        """
        # 逐维度分组的加权相似度
        dims = [
            ("taste", 0.30, self.cos_sim(vector_a[0:31], vector_b[0:31])),
            ("cuisine", 0.25, self.cos_sim(vector_a[32:63], vector_b[32:63])),
            ("ingredient", 0.20, self.cos_sim(vector_a[64:95], vector_b[64:95])),
            ("context", 0.15, self.cos_sim(vector_a[96:111], vector_b[96:111])),
            ("temporal", 0.10, self.cos_sim(vector_a[112:127], vector_b[112:127]))
        ]
        total = sum(w * s for _, w, s in dims)
        return MatchResult(
            overall_score=total,
            dimension_scores={name: s for name, _, s in dims},
            common_cuisines=self.find_intersections(vector_a, vector_b),
            divergence_interests=self.find_divergences(vector_a, vector_b)
        )
```

**检索策略**：

- **粗筛**：通过用户所在城市/活跃区域过滤候选池
- **精排**：向量索引（Milvus/pgvector）IVF + HNSW 近似检索 Top-N
- **重排**：LLM 评估匹配质量，生成匹配解释文本，过滤低质量匹配

**隐私保护机制**：

```
用户数据 → 特征提取 → 味觉向量(128-d float) → 噪声注入(差分隐私ε=1.0)
                                                 ↓
                                         向量索引（仅存储向量+匿名ID）
                                                 ↓
                                          匹配结果 ← 反匿名化 ← 双向确认
```

- 社交匹配全程基于脱敏后的味觉向量，不暴露任何原始记录
- 用户双向确认后才能查看对方的可公开信息
- 用户可随时退出匹配池、删除味觉向量

## 六、Agent 编排与状态管理

### 6.1 编排器（Orchestrator）

```python
class AgentOrchestrator:
    """
    Agent 编排器：负责事件路由、任务分发、Agent 间通信
    """

    def __init__(self):
        self.taste_agent = TasteAgent()
        self.social_agent = SocialAgent()
        self.city_agent = CityAgent()

        # 事件 → Agent 映射
        self.routes = {
            "meal.uploaded":    [self.taste_agent],
            "meal.batch_sync":  [self.taste_agent, self.city_agent],
            "week.cron":        [self.taste_agent, self.social_agent],
            "city.snapshot":    [self.city_agent],
            "user.location":    [self.city_agent, self.taste_agent],
            "match.requested":  [self.social_agent],
        }

    async def handle_event(self, event: AgentEvent):
        agents = self.routes.get(event.type, [])
        tasks = [agent.perceive(event) for agent in agents]
        perceptions = await asyncio.gather(*tasks)

        decisions = []
        for agent, perception in zip(agents, perceptions):
            decision = await agent.reason(perception, agent.memory)
            decisions.append((agent, decision))

        results = []
        for agent, decision in decisions:
            result = await agent.act(decision)
            await agent.reflect(result, agent.memory)
            results.append(result)

        return results
```

## 七、安全与隐私设计

### 7.1 数据分级

| 级别 | 数据 | 存储策略 | 访问控制 |
|------|------|---------|---------|
| L0 原始 | 食物照片、位置信息 | AES-256 加密存储 | 仅用户本人+个人Agent |
| L1 特征 | 味觉向量(128维) | 差分隐私处理后存储 | 本人+匹配系统（脱敏） |
| L2 聚合 | 城市统计指标 | 聚合后存储（k>100） | 公开 |
| L3 输出 | 洞察文本、推荐 | 普通存储 | 本人 |

### 7.2 差分隐私注入

```python
def inject_differential_privacy(
    taste_vector: np.ndarray,
    epsilon: float = 1.0,
    sensitivity: float = 2.0
) -> np.ndarray:
    """
    对味觉向量注入拉普拉斯噪声，满足差分隐私
    - epsilon=1.0: 较强隐私保护
    - sensitivity: 单条数据对向量最大影响
    """
    scale = sensitivity / epsilon
    noise = np.random.laplace(0, scale, size=taste_vector.shape)
    return taste_vector + noise
```

### 7.3 数据生命周期

```
采集 → 特征提取 → 差分隐私 → 向量存储(保留)
  │        │
  │        └─── 原始照片 → 30天TTL → 自动删除
  │
  └─── 位置精确数据 → 7天TTL → GeoHash降级(精确到街区)
```
